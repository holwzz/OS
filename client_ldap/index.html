    Unir un client al LDAP

    Aquesta secció descriu el procediment per integrar els clients al domini LDAP. El procés es compon d'uns quants passos clars i directes, dissenyats per garantir una implementació eficient i sense complicacions.

    Instal·lació requirida

    Abans de començar el procés d’integració al domini LDAP, és fonamental actualitzar el sistema per garantir que es disposa de les versions més recents dels paquets.

    Actualització del Sistema

    Primer, actualitzeu la llista de paquets executant la comanda següent:

    sudo apt update

    Instal·lació dels Paquets Necessaris

    Instal·leu els següents paquets essencials per a la integració LDAP:

    sudo apt install libnss-ldap libpam-ldap nscd

    Durant el procés d'instal·lació, es mostrarà una finestra de configuració en la qual haureu d’introduir els paràmetres adequats per a la connexió amb el servidor LDAP.

    Si la finestra es tanca accidentalment o necessiteu modificar la configuració posteriorment, podeu tornar a obrir-la amb la comanda següent:

    sudo dpkg-reconfigure ldap-auth-config

    Aquesta acció permet reconfigurar els paràmetres del domini LDAP de manera senzilla i eficient.

    Configuració del client LDAP

    Durant la configuració inicial, introduïu l'adreça del servidor utilitzant el format correcte:
    ldap://adreça-del-servidor.

    Important: Si es mostra ldapi:///, aquest format no és vàlid i s'ha de modificar.

    A la finestra de configuració de l'autenticació LDAP, s'ha d'introduir el nom distingit (DN) de la base de cerca. Aquest valor es basa en el domini de l'organització.

    Durant el procés de configuració, assegureu-vos de seleccionar la versió 3 del protocol LDAP per garantir compatibilitat i funcionalitat òptima.

    Per garantir que l'usuari root local tingui permisos d'administrador sobre la base de dades LDAP, seleccioneu Sí en la finestra de configuració corresponent.

    Per garantir l'accés segur a la base de dades LDAP, és necessari habilitar l'autenticació. A la finestra de configuració, seleccioneu Sí quan se us demani si la base de dades LDAP requereix autenticació.

    Per gestionar la base de dades LDAP com a administrador, utilitzeu un compte d'usuari "admin" que ja hagueu creat prèviament. Aquest usuari ha de tenir permisos d'administrador per realitzar tasques de configuració i manteniment. Despres ens demanara la contrasenya corresponent del compte d'administrador LDAP

    En aquesta configuració, s'ha afegit un usuari sense privilegis administratius per a la base de dades LDAP. En entorns reals, és recomanable utilitzar un nom d'usuari diferent i més descriptiu, adaptat a les necessitats de l'organització.

    Per accedir a la base de dades LDAP, introduïu la contrasenya corresponent. Aquesta contrasenya s'utilitzarà per autenticar-se i gestionar l'accés a la base de dades.

    En la configuració del hash per a la base de dades LDAP, deixeu el valor predeterminat com MD5. Aquest mètode de hash s'utilitza per emmagatzemar les contrasenyes de manera segura.

    Modificacio del nsswitch.conf

    Per integrar LDAP al sistema i assegurar que l'autenticació es realitzi mitjançant LDAP en lloc de consultar informació localment, afegiu les línies següents al fitxer de configuració corresponent:

    Configuració common-session

    Per configurar el sistema perquè creï automàticament directoris personals per als usuaris que s'autentiquen mitjançant LDAP, editeu el fitxer /etc/pam.d/common-session amb el següent comandament:

    sudo nano /etc/pam.d/common-session

    Afegiu la següent línia al final del fitxer:

    ion optional pam_mkhomedir.so skel=/etc/skel umask=022

    Aquesta línia assegura que es creï un directori personal per a cada usuari que iniciï sessió, utilitzant la plantilla de directoris /etc/skel i amb permisos definits per l'umask 022. Això facilita la gestió d'usuaris en entorns LDAP.

    Configuració LightMD

    Per permetre als usuaris introduir manualment el seu nom d'usuari durant l'inici de sessió, editeu el fitxer de configuració del greeter i afegiu la següent línia al final:

    greeter-show-manual-login=true

    Aquesta línia habilita una opció al greeter (la pantalla d'inici de sessió) que permet als usuaris escriure manualment el seu nom d'usuari en lloc de seleccionar-lo d'una llista. Això és especialment útil en entorns on es gestionen múltiples comptes d'usuari o en sistemes integrats amb LDAP.

    Verificació de l'Autenticació LDAP

    Per confirmar que el sistema està configurat correctament per autenticar-se mitjançant LDAP, seguiu aquests passos:

    Verificació de l'usuari amb la comanda getent Per comprovar que el sistema està consultant correctament el servidor LDAP, executeu la següent comanda:

    sudo getent passwd

    Si l'usuari que heu afegit al servidor LDAP (per exemple, alu1) apareix en la llista, això indica que la configuració s'ha realitzat amb èxit i que el sistema està integrant correctament l'autenticació LDAP.

    Aquest pas és essencial per assegurar que el sistema reconeix els usuaris definits al servidor LDAP i que la comunicació entre el client i el servidor està funcionant com s'espera.

    Inici de sessió amb l'usuri

    Per assegurar-vos que tots els serveis estan completament operatius i que la configuració LDAP s'ha aplicat correctament, és necessari reiniciar la màquina. Aquest pas garanteix que tots els canvis de configuració s'hagin carregat i que el sistema estigui preparat per autenticar usuaris mitjançant LDAP.

    Per verificar que la configuració LDAP està funcionant correctament, intenteu iniciar sessió utilitzant un usuari definit al servidor LDAP en aquet cas alu1

    Si la configuració LDAP és correcta, el sistema us permetrà accedir amb l'usuari alu1. Aquest pas confirma que el sistema està integrant correctament l'autenticació LDAP i que els usuaris poden iniciar sessió sense problemes.

    Després d'iniciar sessió amb l'usuari LDAP (per exemple, alu1), podeu verificar que esteu autenticat correctament amb la següent comanda:

    whoami
