    Còpia de Seguretat i Automatització a Windows

    Introducció

    En aquesta secció aprendrem a realitzar còpies de seguretat automàtiques a Windows mitjançant un script.

    Primer de tot, afegirem un nou disc a la nostra màquina virtual.

    Formatarem el disc amb el sistema de fitxers NTFS i crearem una carpeta dins del volum per emmagatzemar-hi les còpies.

    Creació de l'Script de Còpia

    A continuació, crearem un script que copiarà automàticament el perfil de l'usuari actual cap a G:\CopiesUsuaris\%USERNAME%.

    @echo off  
mkdir "G:\CopiesUsuaris\%USERNAME%"  
xcopy "C:\Users\%USERNAME%" "G:\CopiesUsuaris\%USERNAME%" /E /H /C /Y

    Explicació dels paràmetres de xcopy:

    • /E : Inclou totes les subcarpetes, fins i tot les buides.

    • /H : Copia fitxers ocults i de sistema.

    • /C : Continua la còpia encara que es trobin errors.

    • /Y : Sobreescriu fitxers existents sense demanar confirmació.

    Resultat esperat:

    Per exemple, per l’usuari higger, es crearà la carpeta:

    G:\CopiesUsuaris\alumne1\

    Automatització de l'Script

    Per executar l'script automàticament a l'inici de sessió, modificarem les polítiques de Windows.

    Procediment:

    Cerca gpedit a la barra d'inici i obre l'Editor de Polítiques de Grup.

    Navega fins a:

    Configuració de l'usuari → Configuració de Windows → Scripts (Inici/Tancament de sessió) → Inici de sessió

    • Fes doble clic a Inici de sessió i després:

    • Clica Afegeix...

    • Fes clic a Navega... i selecciona el fitxer .bat que has creat.

    Un cop configurat l'script, cal executar manualment una actualització de les polítiques de grup.

    Per fer-ho:

    1. Prem Windows + R, escriu cmd, i després prem Ctrl + Shift + Enter per obrir la consola com a administrador.
    2. A la finestra del terminal, escriu la següent comanda i prem Enter:
    gpupdate /force

    Explicació de gpupdate /force

    • gpupdate és l'ordre encarregada de refrescar les polítiques de grup assignades al sistema o a l'usuari.

    • L'opció /force obliga a tornar a aplicar totes les polítiques, independentment de si hi ha hagut canvis o no.

    Per què és important executar-la?

    Normalment, després de configurar scripts d'inici de sessió amb l'Editor de Polítiques de Grup (gpedit), els canvis no es fan efectius immediatament. Amb gpupdate /force, podem aplicar aquestes polítiques al moment, evitant haver d'esperar a un nou inici de sessió o a reiniciar l'ordinador.

    Això resulta especialment pràctic en entorns de proves o pràctiques educatives, on es necessita validar ràpidament els canvis.

    Verificació del Funcionament

    En iniciar sessió amb l'usuari higger, es crearà automàticament la carpeta corresponent i es realitzarà la còpia de seguretat de manera transparent en segon pla.
