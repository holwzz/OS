    Compartició de Recursos amb NFS

    Comparativa: Samba vs NFS

    Característica Samba NFS
    Protocol SMB/CIFS NFS
    Entorn principal Compartició de fitxers en entorns Windows i Linux Compartició de fitxers en entorns Linux/Unix
    Autenticació Basada en usuaris i integrada amb dominis Windows (com Active Directory) Basada en permisos i autenticació a nivell de host (permisos Unix)
    Compatibilitat Compatible amb Windows, Linux i altres sistemes Principalment dissenyat per a Linux/Unix; Windows pot actuar com a client NFS amb configuració addicional
    Casos d'ús Ideal en entorns heterogenis on interactuen Windows i Linux Recomanat per a xarxes d'entorns Unix/Linux, amb alt rendiment en transferències de fitxers
    Rendiment Pot ser menys eficient que NFS en entorns Linux per sobrecàrrega del protocol SMB Generalment més ràpid en xarxes Unix/Linux perquè utilitza RPC directe
    Seguretat Suporta xifratge, ACLs i integració amb Active Directory Depèn de permisos Unix i Kerberos per autenticació segura
    Configuració Requereix configuració addicional per a entorns Linux Més senzill de configurar en entorns Linux
    Gestió d'usuaris Gestió centralitzada mitjançant Active Directory o comptes locals Control d'accés basat en UID/GID i permisos de sistema
    Escalabilitat Adequat per a grans xarxes heterogènies amb integració d’usuaris Més eficient en entorns Linux de gran escala sense Windows

    Instal·lació del Servei NFS en Linux

    Per instal·lar el servei NFS en un servidor Linux amb LDAP configurat, seguim els següents passos

    Instal·lació del servidor NFS:

    sudo apt install nfs-kernel-server

    Comprovació de l'estat del servei:

    sudo systemctl status nfs-server

    Compartir un Directori

    Per compartir un directori, seguim aquests passos:

    Creació del directori:

    sudo mkdir compartida2

    Assignació de permisos:

    sudo chown nobody:nogroup compartida2
sudo chmod 777 compartida2

    Configuració del directori compartit:

    Obrim el fitxer /etc/exports

    I afegim la següent línia:

    /compartida2 *(rw,sync,no_subtree_check)

    Reinici del servei NFS:

    sudo systemctl restart nfs-kernel-server

    Creació d'un fitxer de prova:

    sudo touch /compartida2/fitxer_prova.txt

    Client Windows

    Per accedir al recurs compartit des d'un client Windows, seguim aquests passos:

    Activació del servei NFS 1. Anem a "Programes i característiques" > "Activar o desactivar característiques de Windows".

    1. Activem totes les opcions sota "Serveis per a NFS".

    Mapeig de la unitat de xarxa 1. Obrim l'Explorador de fitxers i seleccionem "Aquest PC" > "Mapejar una unitat de xarxa". 2. Introduïm la ruta al recurs compartit, per exemple: \\IP_SERVIDOR\compartida.

    Verificació de l'accés 1. Creem un fitxer o directori des de Windows i comprovem que apareix al servidor.

    Quan utilitzem la comanda ls -l per verificar si els fitxers o directoris creats des d'un client Windows s'han generat correctament, podem notar que els camps d'usuari i grup apareixen com a números en lloc de noms. Això passa perquè, en sistemes NFS, les identificacions d'usuari i grup es basen en valors numèrics (UID i GID). Si aquests identificadors no tenen una correspondència amb usuaris o grups definits localment al sistema o en un servidor LDAP, es mostren directament com a valors numèrics en lloc de noms reconeixibles.

    Client Ubuntu

    Per muntar un directori compartit via NFS en un client Linux, seguim aquests passos:

    Instal·lació dels paquets necessaris:

    sudo apt install nfs-common rpcbind

    Creació del punt de muntatge:

    sudo mkdir /mnt/nfs
sudo chmod -R 777 /mnt/nfs

    Muntatge del directori compartit:

    sudo mount 10.0.2.15:/compartida /mnt/nfs/

    Verificació del muntatge:

    df -h

    Accés al directori compartit:

    ls /mnt/nfs/

    Creació d'un fitxer de prova:

    sudo touch /mnt/nfs/fitxer_prova_linux.txt

    Observacions sobre Permisos i Usuaris

    En entorns NFS, els permisos es gestionen mitjançant UID (User ID) i GID (Group ID). Si no hi ha correspondència d'usuaris entre el client i el servidor, els fitxers es mostraran amb l'usuari/grup nobody:nogroup. Això es deu a la configuració predeterminada de "root squash" en /etc/exports, que evita que l'usuari root del client tingui privilegis al servidor.

    Amb aquesta configuració, tant el client Windows com el Linux poden accedir i compartir fitxers de manera eficient a través de NFS.

    Perfils Mòbils

    Crearem un directori anomenat "perfils" per guardar els perfils mòbils dels usuaris. Aquest directori estarà assignat a l'usuari "nobody" i al grup "nogroup".

    Establirem els permisos a 777, permetent lectura, escriptura i execució per a tots els usuaris. Tot i que això és útil per a proves, no és aconsellable en un entorn de producció, ja que podria comprometre la seguretat del sistema.

    Per compartir el directori perfils a través de NFS, cal afegir la configuració següent al fitxer /etc/exports:

    /perfils *(rw,sync,no_root_squash,no_subtree_check)
    • rw → Permet tant la lectura com l'escriptura des dels clients NFS.
    • sync → Garanteix que els canvis s’escriguin immediatament al disc per evitar possibles pèrdues de dades.
    • no_root_squash → Manté els privilegis de root dels clients en el servidor. Per defecte, NFS mapeja root com a nobody:nogroup, però amb aquesta opció es desactiva aquest comportament.
    • no_subtree_check → Millora el rendiment desactivant la verificació de subarbres, evitant així problemes d’accés quan es comparteixen directoris sencers.

    Amb aquesta configuració, el directori perfils estarà accessible per als clients NFS amb permisos adequats.

    Després de modificar la configuració, cal reiniciar el servei NFS perquè els canvis tinguin efecte:

    sudo systemctl restart nfs-kernel-server

    Amb això, el directori perfils quedarà disponible per als clients NFS amb la configuració establerta.

    Perfil mobil amb LDAP

    Abans de començar, cal crear un usuari. Per això, he elaborat un fitxer amb extensió .ldif on he especificat els paràmetres següents.

    Un cop creat el fitxer .ldif, el carreguem al directori LDAP fent servir la comanda ldapadd.

    Configuracio Client

    En primer lloc, definirem el directori que s'utilitzarà per allotjar el perfil mòbil i li assignarem els permisos corresponents.

    Seguidament, editarem el fitxer /etc/fstab per configurar el muntatge del directori. Atenció! Qualsevol error en aquest fitxer pot causar problemes greus en el sistema, fins i tot impedir que es pugui iniciar correctament.

    Després de fer els canvis, podem executar la comanda mount -a per muntar totes les unitats especificades a /etc/fstab i comprovar que el muntatge s'ha realitzat correctament.

    Quan iniciem sessió amb l'usuari "xanax", el sistema genera automàticament el seu directori de perfil dins de "/perfil".

    Per acabar, obrim un terminal i verifiquem que l'usuari ha iniciat sessió correctament executant les comandes següents:

    • whoami → Mostra el nom de l'usuari actual.
    • id → Mostra l'UID, GID i els grups als quals pertany l'usuari.
    • pwd → Indica el directori de treball actual.
